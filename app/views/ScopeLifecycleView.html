The $scope object lifecycle has four distinct stages:

  1. Creation
    When a controller or directive is created, Angular creates a new scope for us and passes this on to the new scope when it runs the controller constructor function at runtime.
  2. Linking
    When angular start running, all of the $scope objects are attached or linked to the view. All functions that create $scope objects attach themselves to the view as well. These scopes will register functions that run when things change in the context of the angular app.
  3. Updating
    When the event loop is running, which usually executes on the top-most $scope object (called the $rootScope), every child scope performs itâ€™s own dirty checking. Every watch function is checked for changes. If any changes are detected, then the $scope object will fire the callback.
  4. Destruction
    When a $scope object is no longer needed in the view, the scope itself will be cleaned up and destroyed using a method on the $scope called: $destroy()
